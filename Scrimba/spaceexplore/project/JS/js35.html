Import and Export Statements in JavaScript

JavaScript's import and export statements are used to share code between files, allowing for modular programming. They are part of ES6 modules, which enable you to organize code into reusable components.

1. Exporting in JavaScript

The export statement allows you to make variables, functions, classes, or objects available for import in other files.

Named Exports

Named exports allow you to export multiple items from a module by their names.

// mathUtils.js
export const PI = 3.14159;

export function add(a, b) {
    return a + b;
}

export function subtract(a, b) {
    return a - b;
}

In this example, "PI", "add", and "subtract" are all named exports. You can export multiple items from a single file.

Default Exports

A module can also have a default export, which can be a single value, function, or class. A file can only have one default export.

// greeting.js
export default function greet(name) {
    return `Hello, ${name}!`;
}

Here, the greet function is the default export of the file.

2. Importing in JavaScript

The import statement allows you to bring in functions, variables, or classes from other modules.

Importing Named Exports

To import named exports, use curly braces {} and match the exact names of the exported items.

// app.js
import { PI, add, subtract } from './mathUtils.js';

console.log(PI);             // 3.14159
console.log(add(2, 3));      // 5
console.log(subtract(5, 3)); // 2

You can also rename the imported values using as:

import { add as addition, subtract as subtraction } from './mathUtils.js';

console.log(addition(4, 2)); // 6
console.log(subtraction(5, 3)); // 2

Importing Default Exports

For default exports, simply provide a name for the imported module without curly braces:

// app.js
import greet from './greeting.js';

console.log(greet('Alice')); // "Hello, Alice!"

Importing All Exports

You can import all exports from a module at once using the * syntax:

import * as mathUtils from './mathUtils.js';

console.log(mathUtils.PI);               // 3.14159
console.log(mathUtils.add(4, 5));        // 9
console.log(mathUtils.subtract(10, 3));  // 7

// mathUtils.js

    return a * b;

Then you can import them like this:

// app.js
import multiply, { PI } from './mathUtils.js';

console.log(PI);            // 3.14159
console.log(multiply(3, 5)); // 15

4. Dynamic Imports

Dynamic imports allow you to load modules asynchronously using the import() function, which is useful for code-splitting and on-demand loading.

// app.js
async function loadMathUtils() {
    const { add, subtract } = await import('./mathUtils.js');
    console.log(add(4, 5));     // 9
    console.log(subtract(8, 3)); // 5
}

loadMathUtils();

Conclusion

The import and export statements in JavaScript make it easy to share and organize code. Named exports, default exports, and dynamic imports provide flexibility, allowing you to use code modularly across different files in your project.
