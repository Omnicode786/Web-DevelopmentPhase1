Exceptions in JavaScript

Exceptions are unexpected events or errors that occur during the execution of a program. These events can disrupt the normal flow of code and need to be handled gracefully to prevent the program from crashing. In JavaScript, exceptions are managed using the "try...catch" statement, which allows developers to handle errors in a controlled way.

1. What are Exceptions?

Exceptions typically occur when something goes wrong during the execution of a program, such as:

  • Trying to access a variable that doesn't exist.
  • Calling a function that hasn't been defined.
  • Working with an invalid type or unexpected value.

When an exception occurs, JavaScript stops executing the code and throws an error. If this error is not handled, it can crash the entire program.

2. The try...catch Statement

To handle exceptions in JavaScript, the try...catch statement is used. This structure allows you to "try" running some code, and if an error occurs, it "catches" that error and handles it without stopping the rest of the program.

Syntax:

try {
    // Code that may throw an error
} catch (error) {
    // Code to handle the error
}
  • "try block": Contains the code that you want to execute. If an error occurs inside this block, it is caught and passed to the catch block.
  • "catch block":Contains the code that will handle the error. The error object provides information about what went wrong.

Example:

try {
    let result = someUndefinedFunction(); // This will throw an error
} catch (error) {
    console.log('An error occurred:', error.message); // Handle the error
}

In this example, since "someUndefinedFunction()" does not exist, an error is thrown. The catch block captures this error and logs a message to the console.

3. finally Block

The "finally" block is optional and executes whether or not an exception occurred. It's commonly used for cleanup actions, such as closing resources or resetting variables.

Syntax:

try {
    // Code that may throw an error
} catch (error) {
    // Handle the error
} finally {
    // Code that will run regardless of an error
}

Example:

try {
    let result = someUndefinedFunction(); // Error occurs here
} catch (error) {
    console.log('An error occurred:', error.message);
} finally {
    console.log('This will always run.');
}

Even if an error occurs in the try block, the finally block will still execute.

4. Throwing Your Own Exceptions

You can manually throw your own exceptions using the "throw" keyword. This allows you to raise an error when certain conditions in your code are not met.

Syntax:

throw new Error('This is a custom error message');

Example:

function checkAge(age) {
    if (age < 18) {
        throw new Error('Age must be 18 or older.');
    }
    return 'Access granted!';
}

try {
    let result = checkAge(16);
    console.log(result);
} catch (error) {
    console.log('Error:', error.message); // "Error: Age must be 18 or older."
}

In this example, the "checkAge" function throws a custom error if the "age" is less than 18. The "try...catch" block then catches and handles the error.

5. Error Object Properties

When an exception is thrown, the catch block receives an error object. This object contains useful properties to describe the error:

  • message: A description of the error.
  • name: The type of error (e.g., ReferenceError, TypeError, etc.).
  • stack: A stack trace, showing where the error occurred.

Example:

try {
    let result = someUndefinedFunction(); // Throws an error
} catch (error) {
    console.log('Error name:', error.name);   // "ReferenceError"
    console.log('Error message:', error.message); // "someUndefinedFunction is not defined"
    console.log('Error stack:', error.stack); // Stack trace
}

6. Common JavaScript Errors

  • ReferenceError:Occurs when a variable or function that doesn't exist is referenced.
    • console.log(nonExistentVar); // ReferenceError: nonExistentVar is not defined
  • TypeError: Occurs when a value is not of the expected type.
    • let num = 5;
num.toUpperCase(); // TypeError: num.toUpperCase is not a function
  • SyntaxError: Occurs when there's a syntax error in the code.
    • try {
    eval('hello world'); // SyntaxError: Unexpected identifier
} catch (error) {
    console.log(error.message); // "Unexpected identifier"
}
  • RangeError: Occurs when a number is out of its allowed range.
    • function factorial(num) {
    if (num < 0) throw new RangeError('Number must be positive');
    // Factorial logic...
}
factorial(-5); // RangeError: Number must be positive

7. Nested try...catch Blocks

You can nest "try...catch" blocks when handling multiple errors that may occur at different stages in the code.

Example:

try {
    try {
        let data = JSON.parse('{invalid JSON}');
    } catch (error) {
        console.log('Inner catch:', error.message);
        throw new Error('Parsing error occurred');
    }
} catch (error) {
    console.log('Outer catch:', error.message);
}

In this example, the inner "try...catch" handles a "JSON.parse" error, but then throws another custom error, which is caught by the outer "catch" block.

Conclusion

Handling exceptions in JavaScript using "try...catch" is essential for writing robust code. It ensures that unexpected errors don't crash the entire application, and it provides the developer with tools to gracefully handle and resolve errors. Additionally, with the ability to throw custom errors and handle them in a controlled way, you can write more reliable and predictable code.
