"Object.seal()" and "Object.freeze()" in JavaScript

In JavaScript, objects are mutable by default, meaning you can add, delete, or change properties after the object is created. However, there are situations where you might want to prevent further modifications to an object. JavaScript provides two methods to control the mutability of objects: "Object.seal()" and "Object.freeze()".

1. "Object.seal()"

"Object.seal()" seals an object, preventing new properties from being added and marking all existing properties as non-configurable. However, it does allow the modification of existing property values.

Key Points:

  • Cannot Add New Properties: Once an object is sealed, you cannot add new properties to it.
  • Cannot Delete Properties: Properties cannot be deleted from a sealed object.
  • Can Modify Existing Properties: You can still change the values of the existing properties.
  • Non-Configurable Properties: All properties become non-configurable, meaning you cannot change their property descriptors (e.g., convert a data property to an accessor property).

Example:

const user = {
    name: "Alice",
    age: 25
};

Object.seal(user);

user.name = "Bob";  // Allowed: Existing property can be modified
user.gender = "female";  // Not allowed: Cannot add new properties
delete user.age;  // Not allowed: Cannot delete properties

console.log(user);  // Output: { name: "Bob", age: 25 }

2. "Object.freeze()"

"Object.freeze()" freezes an object, making it immutable. This means you cannot add, delete, or modify any properties of the object.

Key Points:

  • Cannot Add New Properties: No new properties can be added to a frozen object.
  • Cannot Delete Properties: Properties cannot be deleted from a frozen object.
  • Cannot Modify Existing Properties: You cannot change the values of existing properties.
  • Non-Configurable and Non-Writable: All properties become non-configurable and non-writable, meaning you cannot change their values or property descriptors.

Example:

const settings = {
    theme: "dark",
    notifications: true
};

Object.freeze(settings);

settings.theme = "light";  // Not allowed: Cannot modify properties
settings.language = "en";  // Not allowed: Cannot add new properties
delete settings.notifications;  // Not allowed: Cannot delete properties

console.log(settings);  // Output: { theme: "dark", notifications: true }

3. Differences Between Object.seal() and Object.freeze()

Mutability of Properties:

  • Object.seal() allows modification of existing properties but prevents adding or deleting properties.
  • Object.freeze() makes the entire object immutable, preventing any changes to properties.

Configurable Properties:

  • Object.seal() marks properties as non-configurable but still writable.
  • Object.freeze() marks properties as non-configurable and non-writable.

4. How to Check if an Object is Sealed or Frozen

You can check if an object is sealed or frozen using Object.isSealed() and Object.isFrozen() methods.

Example:

const obj = { a: 1 };
Object.seal(obj);

console.log(Object.isSealed(obj)); // Output: true
console.log(Object.isFrozen(obj)); // Output: false

Object.freeze(obj);
console.log(Object.isFrozen(obj)); // Output: true

Conclusion

"Object.seal()" and "Object.freeze()" are useful methods to control the mutability of objects in JavaScript. While "Object.seal()" prevents adding or deleting properties but allows modifications to existing ones, "Object.freeze()" makes an object completely immutable. Understanding the differences between these methods helps in writing more secure and predictable code.
