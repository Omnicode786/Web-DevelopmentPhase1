Exceptions in Python

In Python, exceptions are errors that occur during the execution of a program, interrupting its normal flow. They provide a way to handle unexpected situations gracefully without crashing the entire program.

Types of Exceptions

Python has several built-in exceptions, and users can also create custom exceptions. Common exceptions include:

  • Zero division error: Raised when division or modulo by zero occurs.
  • Type error: Raised when an operation or function is applied to an object of an inappropriate type.
  • value error: Raised when a function receives an argument of the right type but with an inappropriate value.
  • File Not Found Error: Raised when a file operation (like opening a file) fails because the file doesn't exist.

Handling Exceptions:

Exceptions can be handled using the try-except block. The code that might raise an exception is placed inside a try block, and the handling code is placed inside the except block.

Example:

try:
    number = int(input("Enter a number: "))
    result = 10 / number
except ZeroDivisionError:
    print("Cannot divide by zero!")
except ValueError:
    print("Invalid input! Please enter a valid number.")

Explanation:

  • try block: This block contains the code that may raise an exception.
  • except Value Error: If the user enters something that cannot be converted to an integer (e.g., entering text like "abc"), a ValueError is raised.
    • This block catches the error and prints a message to let the user know that the input was invalid, asking them to enter a valid number.
  • except ZeroDivisionError: If the user enters 0, the program raises a ZeroDivisionError. This except block catches that specific exception and prints "Cannot divide by zero!" This prevents the program from crashing and informs the user of the issue.

The "else" and "finally" clauses

  • else: Runs if no exception occurs.
  • finally: Runs regardless of whether an exception occurred or not.

Example:

try:
    f = open('example.txt', 'r')
except FileNotFoundError:
    print("File not found.")
else:
    print("File opened successfully.")
finally:
    print("This runs no matter what.")

Raising Exceptions:

You can raise exceptions manually using the raise keyword.

Example:

x = -5
if x < 0:
    raise ValueError("Negative values are not allowed.")

Conclusion

In Python, exceptions are errors encountered during program execution, managed using try-except blocks to handle specific issues without crashing the program. Common exceptions include ZeroDivisionError for division by zero and ValueError for invalid inputs. The else clause executes code if no exceptions occur, while the finally clause runs regardless of exceptions. Custom exceptions can be created to address unique application errors. This approach ensures robust error handling and improves user experience.
