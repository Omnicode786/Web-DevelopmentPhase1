Scope in Python

In Python, scope determines the accessibility of variables, functions, and objects in different parts of your code. Understanding scope is essential for writing clear and bug-free code. There are several types of scope in Python, including Global scope, Local scope, Enclosing scope, and built-in scope.

1. Global Scope

Variables declared outside of any function or block have global scope. They are accessible from anywhere in the code.

Example:

gobalVar = "I am a global variable"

def displayGlobalVar():
    print(globalVar) # Accessible here


displayGlobalVar() # Output: I am a global variable
print(globalVar) # Accessible here too

2. Local Scope

Variables declared within a function are local-scoped, meaning they are only accessible within that function.

Example:

def displayMessage():
    message = "Hello from inside the function!"
    print(message) # Accessible here

displayMessage() # Output: Hello from inside the function!
print(message) # Error: message is not defined

In this example, the "message" variable is only accessible within the "displayMessage" function.

3. Enclosing Scope

Enclosing scope is the scope that allows inner functions to access variables from the outer (enclosing) functions. The accessibility is determined by the structure of the code, meaning that inner functions can access variables from their enclosing functions because of their physical placement within the source code.

Example:

def outerFunction():
    outerVar = "I am outside!"

def innerFunction():
    print(outerVar) # Accessible here due to enclosing scope


innerFunction()


outerFunction() # Output: I am outside!

In this example, "innerFunction" can access outerVar because it is enclosing inside "outerFunction".

3. Built-in Scope

Built-in scope in Python refers to the scope that contains names and objects that are always available in Python, without the need for any imports. These are defined by Python itself and include built-in functions, exceptions, and other objects.

Example:


Str="Hello World"
print(Str)
print(len(Str))
print(type(Str))
# print, len, type, list etc are all built-in.

LEGB rule:

The LEGB rule in Python is a guideline that defines the order in which Python looks up variables and names. LEGB stands for "Local, Enclosing, Global, and Built-in", which are the four types of scopes Python uses to resolve names. Here's a breakdown of each scope and how Python searches for names in the LEGB order:

      Local Scope:
    • This is the innermost scope.
    • It includes names defined within the current function or block.
    • Python looks here first when trying to resolve a name.
    • If the name is found, Python uses it; if not, it moves to the next scope.
      Enclosing scope:
    • This scope comes into play when there are nested functions (a function within another function).
    • It includes names defined in any enclosing functions, but not the global scope.
    • If the name isn't found in the local scope, Python checks the enclosing scope.
      Global Scope:
    • This scope contains names defined at the top level of a script or module.
    • Variables in the global scope can be accessed from anywhere in the module or script.
    • If the name isn't found in the local or enclosing scopes, Python looks in the global scope.
      Built-in Scope:
    • This is the outermost scope.
    • It contains all the built-in names of Python, such as functions like print(), len(), int(), and keywords.
    • If Python can't find the name in any of the previous scopes, it looks in the built-in scope.

Conclusion

Understanding scope is crucial for managing variables and avoiding conflicts in your Python code. By mastering global, local, enclosing, and built-in scope, you can write more efficient, predictable, and maintainable code.
